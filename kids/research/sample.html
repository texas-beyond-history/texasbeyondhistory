Table of Contents

The table of contents relies both on the table itself (adjacent to this note) and the section headers (interspersed throughout the content below).

Each item in the table (marked by the "Section" designation) must be a link with an href with the form "#NAME_OF_SECTION". The section header must have an id attribute which matches it exactly (i.e. id="NAME_OF_SECTION").

[optional] You can allow visitors to return to the top by adding a link to the (top) like this one in your text. The best use of this is at the end of the Section Headers (see below for examples). In order to make use of it, your table of contents must have a matching anchor tag.

The classes on the div in the table of the contents alter its appearance:

  • "float-right" will align to the right
  • "float-left" will align to the left
  • "half-page" will be half page wide
  • "third-page" will be a third of the page wide
  • "full-page" will be as wide as the page
  • by default the toc will be aligned to the right and half the width of the page

[[Section One: Creating a New Exhibit]]] (top)

  1. Copy the contents of the _boilerplate folder into the root folder of dev.texasbeyondhistory.net
  2. Rename the folder to the name of your Exhibit converted to lowercase words separated by hyphens (e.g. “Cabeza Cooking” becomes "cabeza-cooking")
  3. For each subpage/subsection of the exhibit, duplicate the original [[barebones-index]].html file in the root of the _boilerplate folder.
  4. Rename the main page to index.html
  5. Rename other files to meaningful shorthands of the title of the page (e.g. “About” becomes about.html; “Roots and Fish of Coastal Foodways” becomes coastal.html)
  6. Replace all content in [[BRACKETS]] with content that matches its purpose (see below for details)
  7. Copy sections of html from this page to create the visual effects shown here

Replace Content in Brackets

Note: Find all of the places in the files marked with double square brackets [[ ]]. Delete the brackets and substitute correct values. Make sure you get them all.

  • [[YOUR TITLE]] Traditionally the site has replaced this with just the Exhibit Name (e.g. "Cairn Burials"). For search engine purposes, it would be preferable to also include the page name, like this: Exhibit Name: Page Name (e.g. "Cairn Burials: Stone Cairns of West-Central Texas") going forward. This will set the text for the title line in Google's search results.
  • [[YOUR DESCRIPTION]] This should be a tweet-sized summary of the content of the page. When you google a site, the "description" is what Google will use below the title the page on their results page.
  • [[YOUR PAGE TITLE]] This is the page name/main header text that shows up in big letters at the top of your page (e.g. “Stone Cairns of West-Central Texas”).
  • [[YOUR SUBSECTION LINKS (should be the same in top and footer)]] Links to other pages in the Exhibit.
  • [[EXHIBIT HOME PAGE LINKS]] Complex Exhibits sometimes have a deeply nested structure. These links enable users to get back to the home page. Delete this and the entire div it's inside (labelled pt or pt-r) if you're making a simple exhibit.
  • [[YOUR PAGE CONTENT]] The stuff on your page. (See: the Create Page Content section below for details). Note: Also delete the sample content. It was left in to guide you in writing the html.
  • [[YOUR DATE: 5 March 2021]] Replace this text with the date you published this section.

[[Section Two: Formatting]]] (top)

Highlights

You can set content apart visually by applying highlights.

To do so, just add a class of "highlight" to the div that contains the content you want to highlight. Just like I did on this one.

  • by default the highlight will be aligned like this one
  • "float-right" will slide highlights to the right
  • "float-left" will slide to the left (default)
  • "clear" will be prevent text and images outside of the highlight from overlapping with it. Only use "clear" if you are having problems with overlapping text and image.
  • "half-page" will be half page wide
  • "third-page" will be a third of the page wide
  • "full-page" will be as wide as the page (default)

Other Text Formatting

  • Bold: wrap the text in the <strong> </strong > element
  • Italics: wrap the text in the <em> </em > element. It's short for "emphasis"
  • Align text to the center
  • Align text to the right
  • Orange: add the class "orange-text" to any element. Note: This should only be necessary and desireable in rare situations.
  • Black: add the class "black-text" to any element. Note: This should only be necessary and desireable in rare situations.

[[Section Two: Embedding Images]] (top)

Image Classes

The images on this page are examples of how images can be altered using the image classes. The images themselves illustrate how they work.

  • "gallery-img" all images must have this class.
  • "img-right" will align to the right
  • "gallery-img-half-size" will be half page wide
  • "gallery-img-third-size" will be a third of the page wide
  • by default the image will be aligned to the left and the full width of the page
